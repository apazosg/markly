# US-004 — Subida e historial

## User Story

**Como** usuario,
**quiero** subir mis grabaciones y consultar su transcripción y resumen,
**para** tener el acta procesada y buscarla después.

## Use Cases

### US-004-UC-01: Subida [INFERRED]
**Flujo:** tras parar, la sesión (audio + `notes.csv`) se sube a `POST /sessions`. El backend transcribe y resume de forma asíncrona; el cliente refleja el estado (pendiente/listo/error).

### US-004-UC-02: Historial [INFERRED]
**Flujo:** `history_page` lista sesiones locales y remotas, con búsqueda clásica en cliente por título/etiquetas/fecha.
**AC:**
- **US-004-AC-01:** La búsqueda filtra por título, etiquetas y fecha.

### US-004-UC-03: Transcripción y reproceso [INFERRED]
**Flujo:** `transcript_page` muestra el transcript diarizado + resumen; permite re-analizar (reproceso) y editar metadata (título, etiquetas, nombres de hablantes), que se sincroniza con el backend.

## Functional Requirements

### US-004-FR-001 [INFERRED] — *UC-01*
WHEN el usuario termina una grabación, THE SYSTEM SHALL subir audio + notas al backend y reflejar el estado de procesamiento.

### US-004-FR-002 [INFERRED] — *AC-01*
WHEN el usuario busca en el historial, THE SYSTEM SHALL filtrar en cliente por título, etiquetas y fecha.

### US-004-FR-003 [INFERRED] — *UC-03*
WHEN el usuario edita metadata o re-analiza una sesión, THE SYSTEM SHALL sincronizar el cambio con el backend.

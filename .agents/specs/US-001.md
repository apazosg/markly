# US-001 — Autenticación y cuenta

## User Story

**Como** usuario de Markly,
**quiero** iniciar sesión con mi cuenta,
**para** que mis grabaciones se asocien a mí y se sincronicen.

## Use Cases

### US-001-UC-01: Login [INFERRED]
**Flujo:** `login_page` autentica con Firebase; el JWT resultante se usa en todas las llamadas al backend.
**AC:**
- **US-001-AC-01:** Sin sesión válida, la app muestra el login y no permite subir.
- **US-001-AC-02:** Con sesión, las llamadas al backend llevan `Authorization: Bearer <jwt>`.

### US-001-UC-02: Cuenta y créditos [INFERRED]
**Flujo:** `account_page` muestra el consumo/créditos del mes (desde `/usage/summary`) y permite cerrar sesión.

## Functional Requirements

### US-001-FR-001 [INFERRED] — *AC-01, AC-02*
WHILE no haya sesión Firebase válida, THE SYSTEM SHALL mostrar el login; WHEN la hay, SHALL adjuntar el JWT en cada petición al backend.

### US-001-FR-002 [INFERRED] — *UC-02*
WHEN el usuario abre su cuenta, THE SYSTEM SHALL mostrar los créditos usados/restantes del mes obtenidos del backend.
